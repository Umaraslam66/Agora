# City K — Agent Inspector

A view-only, local web app for inspecting agents in the masked, fictional
"City K" simulation. It renders a schematic map (no real basemap — this is a
masked city, so a real one would be wrong), lets you filter and click
individual agents, and shows each agent's persona-card rewrite, habit
strengths, and BT1-window daily choice timeline, plus a city-wide aggregate
panel.

This app is **read-only**: it has no write path back into the repo's data. It
only ever reads static JSON/GeoJSON files from `public/data/`.

## Status: sample data

`public/data/` currently ships **SAMPLE data**, generated by
`scripts/make_sample_data.mjs`, so the app runs out of the box. It is
plausible but entirely synthetic — 200 invented agents, a fictional 34-zone
city layout, and a hand-tuned aggregate series with a ~27% post-onset drop in
the toll arm and a flat placebo arm.

The **real export** will be produced separately, repo-side (read-only over
`runs/`), and dropped into `apps/inspector/public/data/` with the exact same
file names and shapes described below, replacing these samples. This app's
code should never need to change to pick up the real export — only the data
files change.

## Running it

```bash
cd apps/inspector
npm install
npm run dev        # starts the Vite dev server
```

Other scripts:

```bash
npm run build       # type-checks (tsc -b) and builds to dist/
npm run preview     # serves the production build locally
node scripts/make_sample_data.mjs   # regenerates public/data/ sample files
```

No network access is required or used at runtime — the map is a blank
MapLibre GL canvas with our own GeoJSON layers, no tile server, no CDN fonts,
no basemap.

## Data contract

The app is built against exactly these four files in `public/data/`. Whoever
produces the real export must match these shapes byte-for-byte (field names,
nesting, and value domains) — the app does no schema translation.

### `meta.json`

```json
{
  "onset_day": 39,
  "n_days": 102,
  "warmup_days": 10,
  "tiers": ["T1", "T2", "T3", "T4", "T4_noclaims", "T5"],
  "income_bands": ["low", "mid", "high"],
  "outcomes": ["kept_tunnel", "diverted", "mode_change", "suppressed"]
}
```

### `zones.geojson`

A `FeatureCollection` of ~25-35 zone polygons on abstract coordinates near
`[0,0]`: a north-south water channel through the middle, a "crossing"
corridor (the tolled tunnel) and a "bypass" route around it, both as
`LineString` features.

- Zone polygons: `{ "zone_id": "z01".."z34" }`
- Crossing / bypass lines: `{ "kind": "tunnel" | "bypass" }`

The sample generator also emits one extra `{ "kind": "water" }` polygon for
visual context (the channel itself); this is not required by the contract
but is harmless extra context the map layer knows how to skip or render.

### `agents.json`

An array of ~200 objects:

```json
{
  "id": "a0001",
  "tier": "T5",
  "home_zone": "z07",
  "work_zone": "z18",
  "home_xy": [0.0, 0.0],
  "work_xy": [0.0, 0.0],
  "income_band": "mid",
  "n_cars": 1,
  "pass_holder": true,
  "corridor_traveler": true,
  "rewrite_fired": true,
  "outcome": "diverted",
  "card_before": ["rule strings..."],
  "card_after": ["rule strings..."],
  "habit": [
    { "rule": "short label", "series": [[0, 5.2], [1, 5.4]] }
  ],
  "timeline": {
    "toll": [{ "d": 0, "mode": "car", "fac": "T" }],
    "placebo": [{ "d": 0, "mode": "car", "fac": "T" }]
  }
}
```

Notes:

- `card_after` is `null` when `rewrite_fired` is `false` (no rewrite — before
  and after are identical, shown as an unchanged card, not a diff).
- `home_xy` / `work_xy` jitter is baked in already — the app does not add its
  own jitter.
- `fac` (facility) is `"T"` (tunnel), `"R"` (bypass route), or `null` when
  `mode` is not `"car"`, or the trip is suppressed that day.
- `mode` is one of `"car" | "transit" | "walk" | "none"`.

### `aggregate.json`

```json
{
  "days": [0, 1, "...", 101],
  "tunnel": {
    "toll": [1000.0, "..."],
    "placebo": [1000.0, "..."]
  }
}
```

City-wide daily crossing-corridor volume for both CRN-matched arms. The
aggregate panel in the UI is intentionally **unaffected by the filter bar**
(labeled "all agents" in the UI) — it is a whole-population series, not
computed from the ~200 sample agents in `agents.json`.

## Masking rule (read before adding any string, sample, or comment)

This is a public, masked-city research repo. **No real-world place, agency,
road, or city name may appear anywhere in this app** — code, UI strings,
sample data, comments, git history. The simulated world is always "City K";
its currency is always "credits". If you need a name for anything (a rule
string, a zone, a route), invent a neutral one.

Before pushing any change here, check your diff against the repo's versioned
forbidden-token list:

```bash
# from the repo root
grep -rniF -f <(grep -v '^#' grounding/masking/forbidden_tokens.txt | grep -v '^$') apps/inspector \
  --include='*.ts' --include='*.tsx' --include='*.json' --include='*.geojson' \
  --include='*.mjs' --include='*.md' --include='*.css' --include='*.html'
```

A clean run prints nothing. This app must never import or execute anything
from `agents/`, `world/`, `evaluation/`, `calibration/`, or `grounding/` —
reading `grounding/masking/forbidden_tokens.txt` as plain-text data (as the
command above does) is the one sanctioned exception.

## Project layout

```
apps/inspector/
  scripts/make_sample_data.mjs   # sample-data generator (Node, no deps)
  public/data/                   # data-contract files (sample, see above)
  src/
    types.ts                     # data-contract TypeScript types
    lib/
      colors.ts                  # colorblind-safe (Okabe-Ito) palette
      diff.ts                    # LCS line diff for persona-card rewrites
      useCityData.ts             # fetches the four public/data/ files
    components/
      MapView.tsx                 # MapLibre GL map: zones, routes, agent dots
      FilterBar.tsx                # tier / income / cars / pass / rewrite filters
      AgentDrawer.tsx              # side drawer: summary, diff, habit, timeline
      PersonaDiff.tsx              # before/after persona-card line diff
      HabitChart.tsx               # per-rule habit-strength mini line chart
      TimelineStrip.tsx            # toll-arm / placebo-arm daily choice strips
      AggregatePanel.tsx           # city-wide daily crossing-corridor volume
```

## View-only guarantee

There is no write path in this app: it only performs `fetch()` GET requests
against static files under `public/data/`, and holds all filter/selection
state in React component state (never persisted, never sent anywhere). It
does not read or write anything else in the repository at runtime.
